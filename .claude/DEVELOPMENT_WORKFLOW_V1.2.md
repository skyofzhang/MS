# V1.2 开发工作流程

## 核心原则
**Claude Code = 自主开发者**
- 不需要人类确认每个任务
- 自动编译验证 → 自动修复 → 自动推送
- 只在关键节点通知人类（n8n/Notion）

## 自动化循环

```
1. 读取任务 (从Notion或.task/)
     ↓
2. 编写代码
     ↓
3. 本地编译验证 (dotnet build)
     ↓ 失败? → 自动修复 → 回到步骤3
     ↓ 成功
4. Git Commit & Push
     ↓
5. 更新.task/状态
     ↓
6. 继续下一个任务 (回到步骤1)
```

## 人类介入时机

只有以下情况需要人类介入：
1. **Unity打包** - 需要打开Unity GUI操作
2. **需求不明确** - 需要人类澄清
3. **连续修复失败3次** - 可能需要人类判断
4. **涉及敏感操作** - 如删除文件、修改配置

## n8n通知策略

| 事件 | 通知方式 | 说明 |
|------|----------|------|
| 任务完成 | Notion更新 | 静默记录 |
| 编译错误 | Webhook | 尝试自动修复 |
| 阶段完成 | Notion+摘要 | 可选通知人类 |
| 需要人类操作 | 明确提示 | 如Unity打包 |

## Git提交规范

- `[MS-XXX]` - 功能任务
- `[FIX]` - 修复问题
- `[PROGRESS]` - 进度更新
- `[NEED-HUMAN]` - 需要人类操作
